Descripción general de la base de datos de código abierto Altibase

Altibase es el único base de datos híbrida en la industria que combina a la perfección una en memoria base de datos junto con una en disco base de datos en un solo producto de base de datos utilizando una interfaz uniforme que agiliza el procesamiento de datos para lograr la máxima eficiencia.

Por lo tanto, elimina la necesidad de comprar una base de datos en memoria y una base de datos en disco por separado.

La estructura híbrida permite a Altibase disfrutar de las ventajas del almacenamiento en memoria y en disco mientras oculta sus correspondientes debilidades. Para un alto rendimiento, permite el uso de tablas de memoria y para un almacenamiento económico, permite el uso de tablas de disco.

Se sabe que Altibase es 10 veces más rápido que las bases de datos tradicionales en disco . Es 2 veces más rápido que las otras bases de datos en memoria . Altibase ha prestado servicios a más de 650 clientes en todo el mundo y ha realizado alrededor de 6000 implementaciones en varios dominios de la industria, incluidas 8 empresas Fortune Global 500.

Características del producto

# 1) IMDB ( Base de datos en memoria )

Altibase es una base de datos en memoria con una arquitectura de alta velocidad y fácil de usar. En particular, es muy útil para los casos de uso que exigen un alto rendimiento y una latencia extremadamente baja.

# 2) Fragmentación

Sharding es un tipo de división de bases de datos que divide las bases de datos enormes en partes más pequeñas, más rápidas y fáciles de administrar, conocidas como fragmentos.

Altibase ofrece fragmentación única y de vanguardia de una manera que permite la mejora del rendimiento lineal sin cuellos de botella relacionados con el coordinador. Por lo tanto, mientras se escala, no se produce una degradación del rendimiento, independientemente de la cantidad de servidores que se agreguen.

Altibase admite la fragmentación del lado del servidor y del lado del cliente simultáneamente para mejorar el rendimiento y la compatibilidad.

# 3) Flexibilidad de implementación

Su modo de implementación flexible ofrece tres tipos, es decir Solo en memoria, en disco e híbrido (memoria y disco).

# 4) Altamente disponible

Altibase admite tablas con muchas particiones que pueden transferir datos del disco al espacio de tabla de la memoria y viceversa. También tiene propiedades para especificar el tamaño de los datos de caracteres.

Altibase también proporciona funciones remotas para el procesamiento por lotes. Estas funciones solo se pueden utilizar dentro de los procedimientos almacenados. Otra buena característica que garantiza una alta disponibilidad es la copia de seguridad de instantáneas.

También tiene una utilidad llamada Adaptador JDBC a través del cual cualquier dato modificado en Altibase se puede aplicar a otras bases de datos compatibles con JDBC. También ofrece un búfer de soporte en el que se almacena una cierta cantidad de x-log. Esto ayuda a minimizar la replicación de la pérdida de datos.

# 5) Herramientas y utilidades

Altibase es rico en funciones y características al tener todas las herramientas y capacidades relacionales que requieren las aplicaciones de nivel empresarial. Estas herramientas y utilidades forman parte de los principales jefes como Productividad y Administración, Interoperabilidad y Migración, entre otros.

Admite todas las funciones como proceso multiproceso, LOB, replicación de datos, bloqueo de nivel de fila, vista coherente, recuperación de base de datos, tabla, índice de varias teclas, procedimiento almacenado, función almacenada, secuencia, tabla de cola, auditoría de base de datos, enlace de base de datos , Sinónimo, Porción de tablas, Copia de seguridad definida por el usuario, Copia de seguridad en línea y Vista de supervisión.

# 6) Plataformas compatibles

Altibase es una plataforma múltiple y se puede ejecutar en casi todos los sistemas operativos y plataformas empresariales.

# 7) Interfaces

También es compatible con ANSI SQL-92 y otros SQL ofrecidos por los otros DBMS. Es altamente compatible con la interfaz estándar.

# 8) Lenguajes de programación

En Altibase, se admiten varios lenguajes de programación como C, C +, Java y todos los lenguajes que admiten ODBC y JDBC.

Precios

Altibase es gratuito para utilizar DBMS de código abierto. Puede descargarlo fácilmente desde el sitio web de Altibase.

También se ofrece una suscripción paga para brindar soporte profesional. Las tarifas de suscripción son flexibles y competitivas. Puede contactar al proveedor directamente para obtener una suscripción paga.

Atención al cliente

La asistencia al cliente se proporciona mediante formación y tickets de asistencia. Si se produce una falla repentina inesperada, los administradores de bases de datos de Remote Altibase resuelven los problemas.

También ofrecen servicios de verificación de estado preventivo para confirmar la máxima disponibilidad y rendimiento de su implementación de Altibase.

Ventajas de Altibase Database

Echemos un vistazo a las diversas ventajas de Altibase a continuación:

  • De uso gratuito.
  • Menores costos de mantenimiento.
  • Un DBMS de código abierto que ofrece una funcionalidad de fragmentación de vanguardia.
  • 100% libre de infracción de propiedad intelectual.
  • Ofrece un conjunto de funciones rico y confiable.
  • Fácil migración de habilidades y código SQL.
  • Muy flexible y fácil de usar.
  • Alto rendimiento ((más de 100.000 transacciones por segundo) en comparación con otras bases de datos residentes en disco como Oracle.
  • Baja latencia e indexación eficiente.
  • Tiempos de respuesta predecibles y bajos (escala de microsegundos).
  • Compatible con ACID .
  • Cumple con ANSI SQL.
  • Reduce el costo de hardware y software a medida que obtiene DBMS en memoria y en disco en un solo producto.
  • Al ser relacional, compatible con ACID y SQL, Altibase puede sustituir o trabajar en conjunto con Oracle y otras bases de datos heredadas.
  • La migración de otras bases de datos heredadas a Altibase es significativamente menos complicada. También reduce el DBA y los gastos de implementación.

Limitaciones

Aunque hay varias ventajas de Altibase, también existen ciertas limitaciones.

Son:

  • No se admiten índices globales.
  • No es compatible con el servidor y el cliente para Windows.
  • La función de transferencia de datos de DataPort no es compatible a partir de la versión reciente.
  • Actualmente, no tiene su propia herramienta de administración de GUI como la que tiene Oracle. Sin embargo, puede utilizar cualquier utilidad de terceros si necesita una GUI.
  • No es ideal para productos de datos, pero se puede utilizar para mercados de datos.
  • Todavía no es compatible con el acceso móvil.
